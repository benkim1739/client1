# Scrum Operating Rules

## 1. Scope
This document defines the Scrum process used by the project agents for planning, execution, review, and reporting.

## 2. Agent-Only Execution
- The Scrum process is executed entirely by agents.
- Human intervention is excluded during the execution cycle.
- The Scrum Master coordinates the workflow and reports the outcome at the end of each sprint.

## 3. Backlog Lifecycle
1. The Scrum Master derives product backlog items from the requirements.
2. Product backlog items are prioritized.
3. The Scrum Master selects sprint backlog items for execution.
4. If a problem occurs, the product backlog is revised and the cycle restarts.

## 4. Execution Flow
- Scrum Master selects work.
- Developer agent implements the selected backlog item.
- Developer agent runs unit tests and reports the outcome.
- Scrum Master validates the implementation status.
- Reviewer agent conducts review and integration or acceptance testing.
- Reviewer agent reports results to the Scrum Master.
- Scrum Master prepares a sprint end report and requests confirmation.

## 5. Handoff Requirements
All handoffs must include:
- Objective of the work
- Acceptance criteria
- Relevant constraints
- Expected results
- Current status and blockers

## 6. Reporting Requirements
At the end of each sprint, the Scrum Master must produce a sprint end report covering:
- Completed work
- Blocked work
- Test results
- Review findings
- Recommendation to proceed or stop

## 7. Replanning Rule
If implementation or review reveals unresolved issues, the Scrum Master must revise the product backlog and restart the Scrum cycle.
